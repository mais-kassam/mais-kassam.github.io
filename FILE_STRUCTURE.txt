âš ï¸ CRITICAL: Why _config.yml is Required
==========================================
GitHub Pages uses Jekyll to build your site. Without _config.yml,
Jekyll will try to process EVERY file in your demo projects, including:
- All node_modules folders (thousands of files!)
- Package files, documentation, etc.

This causes:
âœ— Build failures
âœ— Extremely slow deployments (10+ minutes)
âœ— Possible timeout errors

The _config.yml file tells Jekyll: "Skip these folders!"
Result: Fast builds (30 seconds) and no errors.
